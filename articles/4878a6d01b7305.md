## はじめに

書籍『実践Claude Code入門』を読みながら、Dev Container環境でClaude Codeを動かしてみました。本記事はその手順の記録です。

https://gihyo.jp/book/2026/978-4-297-15354-0

## 環境

- OS: macOS Sequoia 15.7.4
- エディタ: VSCode

## Dev Container環境の準備

### ハンズオン用リポジトリのクローン

書籍で提供されているテンプレートリポジトリをクローンします。

https://github.com/GenerativeAgents/claude-code-book-template

### 必要な拡張機能・ツールのインストール

VSCodeに以下をインストールします。

- 拡張機能「Dev Containers」

![Dev Containers拡張機能](https://storage.googleapis.com/zenn-user-upload/30c0a6c7ffeb-20260302.png)

https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers

- Docker（Docker Desktop）

Dockerのインストール方法は書籍のサポートリポジトリでも案内されています。

https://github.com/GenerativeAgents/claude-code-book/blob/main/docs/install-docker.md

### Dev Containerの起動
リポジトリのディレクトリをVSCodeで開き、コマンドパレットから「Dev Containers: Reopen in Container」を選択します。

![](https://storage.googleapis.com/zenn-user-upload/6c95bdbc891b-20260302.png)


しばらく待つとコンテナが起動し、Dev Container環境の準備は完了です。

![Dev Container起動](https://storage.googleapis.com/zenn-user-upload/41452bcfa55d-20260302.png)

:::message
Dev Containerの詳細については、公式チュートリアルも参考になります。
https://code.visualstudio.com/docs/devcontainers/tutorial
:::

## Claude Codeのインストール

Dev Containerのターミナルで以下のコマンドを実行し、Claude Codeをインストールします（Native Install）。

```bash
$ curl -fsSL https://claude.ai/install.sh | bash
```

バージョン確認コマンドを打ち、正常にインストールされているか確認します。

```bash
$ claude --version
2.1.63 (Claude Code)
```

## 認証・初期設定

`claude` コマンドでセッションを起動すると、初期設定のフローが始まります。

```bash
claude
```

:::message
セットアップの詳細は公式ドキュメントも参照してください。

https://code.claude.com/docs/ja/setup
:::

以下の設定を順に進めます。

1. **テーマ選択**: ダークモード / ライトモード
2. **アカウント認証**: Claudeアカウントとの接続
3. **セキュリティ確認**: 注意事項の確認
4. **Terminalセットアップ**: ターミナル環境の設定
5. **フォルダ信頼確認**: 作業ディレクトリの信頼設定
6. **拡張機能**: 必要な拡張機能のインストール

## ヘッドレスモードの動作確認

Claude Codeはヘッドレスモード（非対話モード）での実行もサポートしています。

:::message
公式ドキュメントでは、以前「headless mode」と呼ばれていた機能は現在「CLI」という名称に変更されています。
> CLI は以前「headless mode」と呼ばれていました。`-p` フラグとすべての CLI オプションは同じように機能します。
> https://code.claude.com/docs/ja/headless
:::

### 基本的な使い方

`-p`（`--print`）オプションで、単発の質問を投げることができます。

```bash
claude -p "自己紹介をしてください"
```

![ヘッドレスモード実行結果](https://storage.googleapis.com/zenn-user-upload/77e070a1400e-20260302.png)

https://code.claude.com/docs/ja/cli-reference

# Claude Code について
Claude Code は、Anthropic が開発した **AIエージェント型コーディングツール** です。ターミナル上で動作し、開発者のワークフローに直接統合されます。

## 主な特徴
### 1. ターミナルネイティブ
- CLI（コマンドラインインターフェース）として動作
- エディタに依存せず、どの開発環境でも使える
- シェルの中で自然に作業できる

### 2. エージェント型アシスタント
- ファイルの読み書き・編集
- コマンドの実行（ビルド、テスト、Git操作など）
- コードベースの検索・探索
- Web検索による最新情報の取得

### 3. 主な用途
- **バグ修正**: コードを読み、原因を特定し、修正を提案・適用
- **新機能の実装**: 要件を理解し、計画を立て、コードを書く
- **リファクタリング**: 既存コードの改善
- **コードレビュー**: 変更点の分析とフィードバック
- **Git操作**: コミット、PR作成、ブランチ管理
- **デバッグ**: エラーの調査と解決

### 4. 安全性への配慮
- 破壊的な操作（ファイル削除、force push など）は実行前にユーザーに確認
- パーミッションモードにより、ツールの自動実行レベルを制御可能
- 機密ファイル（`.env` など）のコミットを警告

### 5. ツールシステム
| ツール | 用途 |
|--------|------|
| `Read` | ファイルの読み取り |
| `Edit` | ファイルの部分編集 |
| `Write` | ファイルの新規作成・上書き |
| `Bash` | シェルコマンドの実行 |
| `Glob` | ファイルパターン検索 |
| `Grep` | ファイル内容の検索 |
| `Agent` | サブエージェントによる並列処理 |
| `WebSearch` | Web検索 |
| `WebFetch` | Webページの取得 |

### 6. プランモード
複雑なタスクでは、実装前に **計画を立ててユーザーの承認を得る** フローを取ることができます。これにより、手戻りを防ぎます。

### 7. メモリ機能
プロジェクトごとにメモリファイルを保持し、過去のセッションで得た知見（アーキテクチャ、ユーザーの好み、パターンなど）を次のセッションに引き継げます。

## 基本的な使い方
- **自然言語で指示**: 「このバグを直して」「テストを追加して」など、日本語でも英語でも指示可能
- **`/help`**: ヘルプの表示
- **`/commit`**: Git コミットの作成
- **CLAUDE.md**: プロジェクトルートに配置すると、プロジェクト固有の指示を永続的に設定可能

## モデル

現在、Claude Code は **Claude Opus 4.6**（最新・最高性能モデル）で動作しています。サブエージェントには用途に応じて Sonnet や Haiku も選択可能です。
