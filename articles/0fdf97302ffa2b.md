# LT1: Gradle Build Scriptの第0歩:仕組みから書き方を理解する（[@Udomomo](https://x.com/Udomomo)）

@[docswell](https://www.docswell.com/s/Udomomo/ZDWRW6-2025-12-23-201201)

- [Gradle](https://gradle.org/) のビルドスクリプトの書き方だけでなく、その背後にある内部の仕組みや全体像を理解することを目指した発表
- `build.gradle.kts` などの設定ファイルは DSL（ドメイン特化言語）で記述されており、内部的には [Gradle API](https://docs.gradle.org/current/javadoc/) を呼び出してタスクの取得や更新を行っていることが解説された
- Gradle のビルドは **初期化（Initialization）** / **構成（Configuration）** / **実行（Execution）** の 3フェーズに分かれており、特に構成フェーズでタスク同士の依存関係（タスクグラフ）を構築することが重要
- 複雑なビルドにおいてパフォーマンスを向上させるため、タスクのインスタンスをすべて生成するのではなく、参照のみを持つ仕組み（[`TaskProvider`](https://docs.gradle.org/current/userguide/task_configuration_avoidance.html) や Configuration Avoidance API など）を利用して、必要な時にだけタスクを生成することが推奨される
- 普段「呪文」のように扱いがちなビルドスクリプトも、DSL → API → ライフサイクルという視点で読み解くことで、書き方そのものへの納得感が一段深まる、という気付きが共有された

# LT2: 2025年セキュリティニュース振り返ろう（[@mikene_koko](https://x.com/mikene_koko)）

- トレンドマイクロが集計・報告したインシデント件数（501件）をもとに、今年発生したセキュリティインシデントの事例や攻撃の傾向を振り返る発表
- 攻撃種別の1位は **不正アクセス**、2位は **ランサムウェア**
- 事例として、高校生が AI を悪用して攻撃した 快活クラブの不正アクセス事件や、アサヒグループ、アスクル など大手企業を狙ったランサムウェアの被害が紹介された
- クレジットカードの入力画面を改ざんして情報を抜き取る **Webスキミング** の手法も挙げられた
- 侵入経路としては VPN 機器などのネットワークの脆弱性を突くケースが多く、全体の **約8割** を占める
- ターゲット企業に直接攻撃するのではなく、セキュリティの手薄な子会社や提携企業を踏み台にする **サプライチェーン攻撃** が目立っていることも指摘
- 対策として **2段階認証（MFA）** を徹底することの有効性や、完全に侵入を防ぐことは難しいため、**侵入されることを前提とした復旧力（レジリエンス）** を高めておくことの重要性が語られた
- アスクル社が公開した被害報告書は、感染経路から再発防止策まで詳細に開示されており、他企業にとっても必読の有益な資料であると強く推奨された

# LT3: 非同期処理をちゃんと理解したい（ハム）

@[docswell](https://www.docswell.com/s/4699248/52QD44-2025-12-18-215832)

- 初心者向けに、同期処理と非同期処理の違いや、非同期処理の本質について解説した発表
- 並行処理（Concurrency）と 並列処理（Parallelism）の違いを整理した上で、同期処理は待ち時間が発生した際にスレッドをブロックしてしまうのに対し、非同期処理はスレッドをブロックせずに別の処理を進められることが説明された
- 非同期処理の最大のメリットは **「CPU を暇にさせない（遊ばせない）こと」** であり、単に時間のかかる重い計算処理ではなく、API 通信やデータベース操作など、**明確な待ち時間が発生する処理**に導入すると効果的
- 非同期処理の本質は単に「同時実行しているように見せること」ではなく、**「待ち時間にスレッドをブロックせず、CPU を有効活用すること」** であると結論づけられた
