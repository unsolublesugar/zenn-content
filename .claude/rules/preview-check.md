# Local Preview Check

`articles/`・`books/` 配下のコンテンツを新規作成・編集した後、コミット前に必ずローカルプレビューでの目視確認を経ること。

## 手順

1. 自動レビュー（`.claude/rules/auto-review.md`）が完了したタイミングで、Claude は `npx zenn preview` をバックグラウンドで起動する
2. プレビュー URL（通常 http://localhost:8000 ）をユーザーに案内し、対象記事・ブックの表示確認を依頼する
3. ユーザーから明示的なOK（「OK」「問題なし」「コミットして」など）が得られるまで、`git commit` / `git push` / `gh pr create` には進まない
4. 修正指示があれば反映し、再度プレビュー確認を依頼する

## 注意

- すでにプレビューサーバーが起動している場合は重複起動しない
- 作業終了時にバックグラウンドのプレビュープロセスを停止する
- ユーザーが明示的に「プレビュー不要」と指示した場合はスキップしてよい
