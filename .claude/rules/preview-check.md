# Local Preview Check

`articles/`・`books/` 配下のコンテンツを新規作成・編集した後、コミット前に必ずローカルプレビューでの目視確認を経ること。

## 手順

1. 自動レビュー（`.claude/rules/auto-review.md`）が完了したタイミングで、Claude は `npx zenn preview` をバックグラウンドで起動する
2. **起動ログから実際のポート番号を確認** し、対応するプレビュー URL をユーザーに案内する
   - デフォルトは http://localhost:8000
   - ポート8000が使用中の場合、`zenn preview` は自動的に空きポート（8001 など）へフォールバックするため、起動ログ（`👀 Preview: http://localhost:<port>`）の値を必ず読み取って案内すること
   - ログがまだ出力されていない場合は `lsof -i :8000-8010` などで実ポートを特定する
3. 案内したURLで対象記事・ブックの表示確認を依頼する
4. ユーザーから明示的なOK（「OK」「問題なし」「コミットして」など）が得られるまで、`git commit` / `git push` / `gh pr create` には進まない
5. 修正指示があれば反映し、再度プレビュー確認を依頼する

## 注意

- すでにプレビューサーバーが起動している場合は重複起動しない
- 作業終了時にバックグラウンドのプレビュープロセスを停止する
- ユーザーが明示的に「プレビュー不要」と指示した場合はスキップしてよい
