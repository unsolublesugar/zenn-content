# 自動レビュー

記事やブックの本文に対する編集作業が一区切りついたタイミングで、以下の8観点でレビューを自動実施すること。ユーザーからの明示的な `/review` 呼び出しを待たずに行う。

## レビュー観点

1. **技術的正確性**: コード例・コマンド・技術用語・バージョン情報の正確さ
2. **文章構成**: 論理的な流れ、セクション順序、導入→本題→まとめの構成
3. **校閲**: 誤字脱字、表記ゆれ、助詞の誤用、冗長表現、Front Matterの妥当性
4. **論理性**: 主張と根拠の対応、論理展開の飛躍の有無
5. **実用性**: 読者にとっての有用性、手順の再現可能性
6. **読みやすさ**: 文の長さ、段落・リスト・コードブロックの使い分け
7. **独自性**: 著者ならではの経験・知見、公式ドキュメントの焼き直しでないか
8. **明確性**: 想定読者にとっての理解しやすさ、曖昧な表現の有無

## 出力形式

- 各観点ごとに問題の重要度（高/中/低）を付けて一覧化
- 問題がない観点は「問題なし」と記載
- 最後に総合評価を5段階で提示
